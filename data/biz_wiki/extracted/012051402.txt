ThinkSystem SR950 V3

专为要求严苛的工作负载量身打造

ThinkSystem SR950 V3 服务器专为繁重的任务关键型工作负载而设计，例如 SAP HANA、应用程序、数据库、大数据、业务分析、ERP 和 CRM 应用程序以及虚拟化。

这款功能强大的 8U 服务器由第八个第 4 代英特尔®至强®可扩展处理器系列 CPU 组成，适用于计算密集型工作负载，性能较之于第一代系统提高了 172%。

极致性能与可靠性

核心业务工作负载要求系统提供极致的性能与持续的可用性。随着 CPU、内存、存储和 I/O 性能的提高，SR950 V3 扩展了行业标准技术以建立新的性能阈值。

ThinkSystem SR950 V3 集成了八个第 4 代英特尔®至强®可扩展处理器，在 8U 机架空间中支持高达 32TB 的内存和 480 个处理内核。该服务器具有多级别韧性来保护数据，旨在确保持续运行并提供“始终在线”的可靠性。

领先的 SAP 解决方案

ThinkSystem SR950 V3 服务器专为实现最高性能而设计，是要求最苛刻的 SAP HANA 情景部署的理想选择，可提供卓越的功能和更高的业务价值。该服务器可以帮助企业更快制定更明智的决策，即使数据负载不断增加也是如此。与 ThinkSystem SR950 相比，SR950 V3 具备如下优势：

- SAP 用户增加 2 倍
- 比 SAP HANA DB 大 33%
- 带宽增加 63%，可支持更大的 SAP HANA 设备

通过将节点联网，可以轻松建立多节点、横向扩展设置，从而实现更大的 SAP HANA 内存容量。通过这种模块化方法，您可以扩展基础架构以及 SAP HANA 环境。

此外，Lenovo 解决方案通过集成高可用性和自动故障转移功能，摒弃了经常需要的 SAN 存储。

高效的部署、管理和安全性

所有 ThinkSystem 服务器都有一个名为 Lenovo XClarity Controller 的内部管理引擎，旨在标准化、简化和自动化核心服务器管理活动。通过 XClarity Controller，可能会向 Lenovo XClarity Administrator、VMware vCenter 和 Microsoft System Center 公开警报。您可通过这些警报在潜在问题发生之前采取必要的步骤，从而提高服务器的正常运行时间和应用程序可用性。

平台固件弹性 (PFR) 和可信平台模块 (TPM) 2.0 也包含在信任根 (RoT) 模块中，以进一步保护和验证系统。这些模块通过发现未经授权的固件更新，将损坏的镜像恢复到已知的安全镜像，并监控固件以确保其未被篡改，从而帮助防止对重要的平台子系统进行未经授权的访问。

常见问题

Q: ThinkSystem SR950 V3 支持多少颗 CPU？
A: ThinkSystem SR950 V3 支持八个第 4 代英特尔®至强®可扩展处理器。

Q: 该服务器的最大内存容量是多少？
A: ThinkSystem SR950 V3 最高支持 32TB 内存。

Q: SR950 V3 相比第一代系统，性能提升了多少？
A: ThinkSystem SR950 V3 在计算密集型工作负载下的性能较第一代系统提高了 172%。

Q: SR950 V3 是否专为 SAP HANA 优化？有哪些具体优势？
A: 是的，SR950 V3 是专为要求最苛刻的 SAP HANA 部署设计的；相比上一代 SR950，它支持的 SAP 用户数量增加 2 倍、SAP HANA 数据库容量增大 33%、带宽提升 63%，可支持更大的 SAP HANA 设备。

Q: 该服务器是否支持横向扩展的 SAP HANA 部署？
A: 是的，通过将多个 SR950 V3 节点联网，可轻松构建多节点、横向扩展的 SAP HANA 设置，从而实现更大的内存容量和更灵活的基础架构扩展。

Q: SR950 V3 是否需要外置 SAN 存储才能实现高可用性？
A: 不需要。Lenovo 解决方案在 SR950 V3 中集成了高可用性和自动故障转移功能，可替代传统上必需的 SAN 存储。

Q: Lenovo XClarity Controller 在 SR950 V3 中起什么作用？
A: Lenovo XClarity Controller 是内置管理引擎，用于标准化、简化和自动化核心服务器管理活动；它可向 Lenovo XClarity Administrator、VMware vCenter 和 Microsoft System Center 发送警报，帮助在潜在问题发生前主动干预，提升系统正常运行时间和应用可用性。

Q: SR950 V3 如何保障平台固件安全？
A: SR950 V3 配备平台固件弹性 (PFR) 和可信平台模块 (TPM) 2.0，共同构成信任根 (RoT) 模块；该模块可检测未经授权的固件更新、将损坏的固件镜像自动恢复至已知安全版本，并持续监控固件完整性，防止对关键平台子系统的未授权访问。
